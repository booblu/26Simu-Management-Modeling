## 一、课程定位与目标

### 1.1 课程名称与副标题

- **课程名称**：Agentic Protocol Modeling & Simulation Studio（代理协议建模与仿真工作室）
- **副标题**：用仿真审计 AI Agent 在复杂协议与跨协议生态中的策略、风险与治理

### 1.2 与《管理建模与仿真》的关系

- **判题器**：只要项目回答的是——**在资源约束下（资本/流动性/额度/算力/权限/时间），如何设计机制或策略，并用仿真比较其在不确定与动态环境中的绩效（收益/效率/风险/稳定性/合规）**——即属于管理建模与仿真。
- 链上协议 = 可编程组织与机制；AI Agent = 新的执行者/决策者。课程重点：**机制设计 + 策略对比 + 风险治理**。

### 1.3 课程“操作系统”

- **节奏**：8 周，每周半天（4 节课），共 32 节课。
- **项目节奏**：每 2 周完成一个**完整项目**（最多 4 周一个项目），共 4 个项目。
- **参与方式**：每人独立完成（超级个体），所有成果上传 GitHub，并提交**可供 AI 自动验证与打分的交付物**（CI 门禁 + grade.json）。
- **课堂形态**：讲解（Instructor Brief）+ 现场演示（Live Build）+ 实做（Studio）+ 分享与问答（Show & Ask）；每两周一次 Kickoff 半天 + Audit & Demo 半天。

### 1.4 核心能力目标

学生结课后应能：

1. 把**协议**抽象为可计算系统：状态、事件、不变量、边界。
2. 写出**不变量/边界/可复现**的测试，并在 CI 中自动运行。
3. 设计情景（scenarios），一键产出 `metrics.csv` 与图表，用证据回答：**协议/策略在什么情景下最脆弱、什么情景下最稳**。
4. 在单协议、跨协议、多 Agent 博弈等层级上做**策略对比与机制设计**，并输出可复现证据链。

## 三、项目 1（第 1–2 周）：协议的建模与仿真

### 3.1 目标与边界

- **目标**：把 Uniswap v2 抽象成**可计算系统**（状态 + 事件 + 不变量），写出不变量测试，设计至少 6 个情景，一键产出 metrics 与图，用证据回答“最脆弱/最稳”情景。
- **不要求**：链上开发、真实交易/投资分析；不模拟链上环境、MEV、多跳、真实用户博弈。

### 3.2 系统边界与状态（教师讲解要点）

- **模拟范围**：单池（reserve_x, reserve_y）、Swap（x↔y）、Fee（留池）、可选最简 LP（add/remove）。
- **状态变量**：  
  - 核心：`reserve_x`, `reserve_y`, `fee_rate`, `k = x*y`（派生）。  
  - 派生：`price_mid`, `trade_price`。  
  - 可选：`lp_total_shares`, `lp_shares`。
- **事件—状态更新—硬规则**（简要）：

| 事件 | 输入 | 状态更新 | 硬规则 |
|------|------|----------|--------|
| swap x→y | dx | x↑, y↓, k 变化 | 储备非负；输出有界；fee>0 时 k 非降 |
| swap y→x | dy | y↑, x↓, k 变化 | 同上（对称） |
| add/remove（可选） | ax,ay 或 shares | x,y,lp_shares | 份额守恒/按比例（最简） |

### 3.3 不变量与测试（必做 I1–I5，建议 I6–I8）

- **I1** 储备非负（x,y ≥ -eps）。  
- **I2** 输出有界（no-drain）：dy ≤ y_before，dx ≤ x_before。  
- **I3** 边界：dx=0 ⇒ dy=0 且状态不变（y 方向同理）。  
- **I4** 手续费一致性：同输入下 amount_out(fee>0) ≤ amount_out(fee=0)。  
- **I5** 可复现：固定 seed/交易脚本，结果一致（或 eps 内）。  
- **I6** 滑点单调性：大单平均成交价更差。  
- **I7** Constant product：fee=0 时 k 近似恒定；fee>0 时 k 非降。  
- **I8** 状态更新一致：输入全量入池（x_new = x + dx），手续费留池。

### 3.4 情景与输出

- **至少 6 个 scenarios**：2×2 无冲击（High/Low 流动性 × Low/High 强度）+ 至少 2 个带冲击（方向覆盖 x→y 与 y→x）。  
- **统一**：steps=2000，fee_rate=0.003，direction_bias=0.5；Low/High 强度用 trade_probability + size_levels 区分。  
- **输出**：`results/metrics.csv`（含 scenario_name, seed, invariant_violations, min_reserve_x/y, nan_inf_count, avg_price_mid, fee_rate 等）；`results/figures/` 至少 2 张图（price_mid_timeseries, k_timeseries）。  
- **README**：一页 memo，结论 + 证据指向（metrics/figures/tests）。

### 3.5 阅读与工具（对应阅读清单）

- **协议必读（A）**：Uniswap v2 官方文档（How it works, Swaps, Fees, Pricing, Common Errors）+ 白皮书节选。  
- **验证与可复现（B）**：pytest 入门/断言/fixtures、Python random seed、GitHub Actions 跑 Python 测试。  
- **实验输出（C）**：pandas CSV、matplotlib 基本画图与 savefig。  
- **选读（D）**：Uniswap v2 入门解读、v2-core 仓库结构、Hypothesis、SALib/Mesa（后续项目）。  

**最短路线**：A1–A6 + B7–B11 + C12–C13。

### 3.6 交付物与验收（L2）

- **spec.yaml**：课程/项目/版本、决策问题、scope、state_variables、protocol_model、randomness_and_scenarios、verification_plan、outputs。  
- **experiments/scenarios.yaml**：6 个标准情景（S1–S6）。  
- **tests/test_invariants.py**：I1–I8（至少 I1–I5）。  
- **src/simulator.py** 最小接口：`get_amount_out`, `swap_x_for_y`, `swap_y_for_x`。  
- **CI**：pytest 全绿；`python -m experiments.run_all` 生成 metrics.csv 与 figures。  
- **可选**：grade.json（tests/scenarios/artifacts 等机器可验证字段）。

## 四、项目 2（第 3–4 周）：基于单协议的简单策略

### 4.1 目标

- 在**单协议**上做**策略对比**（baseline vs candidate），证明“策略实验”能力，避免只在一个情景上“赢”。

### 4.2 讲解要点

- 策略空间怎么写进 spec（杠杆、阈值、频率、风险上限）。  
- 反过拟合：策略须在情景集合上鲁棒。  
- 敏感性分析最小版：参数扫 + 结论边界。

### 4.3 典型策略题

- AMM：被动持有 vs 再平衡/撤出阈值/动态区间（简化）。  
- 借贷：固定抵押率 vs 动态补仓/止损/清算前预警。  
- 闪电贷可作为“压力情景注入器”（教学用，不要求真实链上交易）。

### 4.4 必交付与验收

- **必交**：`policies.py` 至少 2 个策略；多情景×多种子批量实验；metrics 含收益/成本/风险；README 写明“推荐策略”与“翻车条件”。  
- **自动验收**：candidate 在 ≥X% 情景中不劣于 baseline；尾部风险（最大回撤/破产概率）不显著恶化；输出含 trade-off 图（收益-风险-成本）。  
- **可选加分**：策略成本（gas/滑点）、执行错误率（Agent 误判）入模。

## 五、项目 3（第 5–6 周）：跨协议策略

### 5.1 目标

- **跨协议**路由与组合：资源在多个协议/池子间分配；理解耦合与级联风险。

### 5.2 讲解要点

- 组合系统状态与耦合（一协议冲击如何传导到另一协议）。  
- 实验设计：情景注入优于“拟合历史”（尤其两周周期）。  
- 系统韧性指标：最大损失、恢复时间、崩溃概率。

### 5.3 必交付与验收

- **必交**：simulator 支持 ≥2 个协议模块（模块化）；routing_policy（何时迁移、迁移多少）；至少一类“冲击传播”情景；README 回答跨协议收益来源、新增风险、缓解机制。  
- **自动验收**：模块可单独运行；冲击情景类型×强度覆盖；candidate 在冲击下韧性不弱于 baseline（可允许收益略低但风险更低）。  
- **可选加分**：治理成本入模；策略选择地图（不同风险偏好下选不同策略）。

## 六、项目 4（第 7–8 周）：多 Agent 博弈

### 6.1 目标

- 把市场/协议生态视为**多主体系统**，策略表现依赖“他人行为”；从“我怎么赚钱”上升到“规则怎么改才能降低系统性风险”。

### 6.2 讲解要点

- ABM 最小骨架：主体状态、规则、交互、事件队列。  
- 博弈用“策略实验”比较：均衡趋向、拥挤效应、踩踏风险。  
- 机制设计视角：费用规则、清算惩罚、限速、抽检审计等。

### 6.3 必交付与验收

- **必交**：至少 3 类 Agent（其中 1 类为对抗/套利/清算）；至少 2 套机制/规则对比；系统级指标：稳定性（崩溃概率/大幅波动）、公平性/集中度、效率（成交量/资金利用率等）。  
- **自动验收**：多种子下统计性质稳定；对抗者在 baseline 下能造成可观影响；机制改造后系统风险改善、效率损失可接受。  
- **可选加分**：AI Agent 不可靠性（误动作/误判概率）；审计预算约束（抽检越多越安全但越贵/越慢）。

## 七、统一 Repo 结构与自动验收

### 7.1 标准目录（四个项目一致）

```
project/
  README.md              # 一页决策 memo + 复现说明
  spec.yaml              # 机器可读：决策、KPI、边界、假设、实验计划
  src/
    simulator.py         # 协议/环境仿真
    policies.py          # baseline + candidate（P2 起）
    metrics.py           # 确定性指标计算
  experiments/
    scenarios.yaml
    run_experiments.py   # 一键 -> results/
  results/
    metrics.csv
    figures/
  tests/
    test_invariants.py   # 不变量/边界
    test_reproducibility.py
  grade/
    grade.py             # 生成 grade.json
    rubric.yaml
    grade.json           # CI 产出
  .github/workflows/ci.yml
  requirements.txt
  （可选）Dockerfile、PROMPTLOG.md
```

### 7.2 AI 可自证自评原则

- **分数主干**由确定性程序产生（CI + grade.py → grade.json）。  
- **AI grader**：基于 grade.json 与证据路径，检查 memo 与证据一致性、边界与风险表述，并给出解释性点评；不依赖“主观感觉”打分。

### 7.3 门禁与证据

- **硬门禁**：tests 全绿、可复现（固定 seed）、运行时间上限（如 120s）。  
- **证据生成**：跑情景 → metrics.csv / figures → grade.py → grade.json。

## 八、8 周每周半天安排（含讲解、实践、分享与问答）

### 固定节奏（每周 4 节）

1. **Instructor Brief（讲解）**：45–60 min — 本周唯一关键概念 + 常见坑 + 验收标准。

4. **Show & Ask（分享+问答）**：30–45 min — 3–5 人闪电分享（坑与验证）；公共 Q&A。

## 九、八周讲授单元（你“讲什么”的系统化）

1. **建模合同与可验收性** — 什么叫“模型做完”：CI 全绿 + 证据链。  
2. **协议作为动态系统** — 状态、事件、守恒、不变量（以 AMM x*y=k 为例）。  
3. **随机性建模** — 交易流/价格/冲击（情景 > 拟合）。  
4. **策略实验设计** — 避免过拟合、情景集合、对照与消融。  
5. **敏感性与鲁棒性** — 参数扫、尾部风险、破产概率。  
6. **跨系统耦合与级联风险** — 传播、反馈、恢复时间。  
7. **多主体与机制设计** — 拥挤、对抗、规则改造。  
8. **治理与审计** — 权限、抽检、限速、回滚（可计算机制）。

## 十一、安全与边界（建议写进 Syllabus）

- 所有实验为**离线仿真/合成或公开数据**；不要求、也不允许真实资金交易或连接真实钱包。  
- 课程产出是**建模与仿真证据链**，不构成投资建议。  
- 评价重点：模型结构、验证、实验设计、鲁棒性、治理机制，而非“赚得最多”。
