# 1.3 案例演练：三甲医院急诊分诊系统的状态空间降维
*(Case Study Wargaming: State-Space Dimensionality Reduction of a Hospital Emergency Triage System)*

---

抛开抽象的金融与代码，为了让你的大脑长出“二阶建模者”的肌肉，我们本节将使用一个极度世俗、人人秒懂，却又包含极高维排队博弈论与资源挤兑的复杂场景：**一家市级三甲医院的急诊室网络**。

请试想你被聘请为这家陷入连环医疗事故与极度拥堵灾难的医院的 CEO（或总架构师）。你的任务是重新构建它的数字孪生模型（Digital Twin），通过仿真来找到破局点。

如果你使用传统的“管理流派”，你会写一堆诸如“优化护士排班体系”、“增强医患人文关怀”、“缩短检测报告延迟”等充满文字颗粒度的废话。而这种废话在计算机面前毫无意义，根本无法用来承载量化测算。

现在，拿起你上一节缴获的“四把状态机手术刀”，让我们对急诊室进行物理切割。

---

## 🔪 步骤一：剥离实体与资源 (Entities & Resources)

* **实体 (Entities)**：我们在这个仿真里关心的唯一活动流是“病人”。不再区分病人的性别与职业，我们只关心其在诊断系统中的“分诊级别（Triage Level：红、黄、绿）”。
* **资源上限 (Resources Capacities)**：医院的物理瓶颈到底卡在哪里？
  - 分诊台护士：3 名。
  - 急救创伤床位 (红区)：5 张。
  - 普通观察床位 (黄/绿区)：20 张。
  - 缝合手术室医生：2 名。

我们战略性地抛弃了“输液椅数量”、“家属等待区面积”等冗余噪音，**这就是第一性降维**。

---

## 🔪 步骤二：定格状态快照字典 (State Snapshots)

什么是急诊室在计算机存储池里的“全息切片”？
假设我们冻结了晚上 10:35:12 秒的时空。在这一刻，急诊室的状态向量不包含任何悲欢离合，它仅仅是一串数字的集合：

**系统当前状态 $S_t$** = 
- `Queue_Triage`: 分诊台外正在排队的人数（整型）。
- `Red_Beds_Occupied`: 当前被占据的红区病床数（0 ~ 5）。
- `Docs_Available`: 当前空闲的外科医生数（0 ~ 2）。
- `Patients_Waiting_for_Surgery`: 躺在黄区病床上但急需手术医生介入的黄码病人数。

这组变量的跳动轨迹，构成了整个系统演化的时空图谱。

---

## 🔪 步骤三：编纂外生事件扳机 (Event Triggers)

是什么打破了系统死寂的僵局？
* **外生到达事件 (Arrival Event)**：
  - 救护车刺耳的鸣笛（“红码病人到达事件”）。
  - 醉汉被同伴搀扶进门（“黄码/绿码病人到达事件”）。
  由于外部世界的极度混沌，你必须从该医院过去五年的历史数据库中，提取出这种到达事件的**柏松分布概率（Poisson Distribution）**。比如：凌晨 2 点到 3 点，平均每小时发生 0.2 起创伤到达事件。机器将利用这个数学抛物线，在后台扔骰子。

* **内生结束事件 (Completion Event)**：
  - 医生擦着汗走出手刀室（“手术完成事件”），该事件在手术开始时便触发了一个“倒计时雷管（如耗时 服从 45~90 分钟的均匀分布）”，倒数归零即引爆该事件，将 1 名医生转移回 `Docs_Available` 状态。

---

## 🔪 步骤四：重写高压策略响应 (Policy / Action Logic)

这里是真正的生死修罗场。当极压情况发生（红码事件爆发，且系统核心资源归零），这台无情的状态机该如何执行判定？这就是高管的最高决策体现。

* **事件爆发**：深夜 11 点，高速连环车祸，突然触发了超高并发的 8 起【红码到达事件】。
* **状态冲撞**：此时 `Red_Beds_Occupied == 5` ，物理上限已被锁死填满！

**传统流程图的瘫痪**：流程图在这里就会报错断裂，因为箭头无路可指。
**计算状态机的铁血策略 (Policy Assertions)**：
你作为高管，写入了这段冷血的逻辑门代码：
```text
IF Event == Red_Patient_Arrive:
    IF Red_Beds_Occupied < 5:
        Red_Beds_Occupied += 1
    ELSE IF Yellow_Beds > 0 (强制腾退黄码区间):
        Discharge_Green_Patient_From_Yellow_Bed()
        Yellow_Beds -= 1
        Pseudo_Red_Beds += 1
    ELSE:
        // 极压熔断防线触发
        Trigger_System_Failure_Event(Divert_To_Other_Hospital)
```

**案例大成**：
一旦这四大支柱被你抽丝剥茧地定义进计算机。这套医疗急诊乱象，就蜕变成了一台可被 10万次蒙特卡洛轰炸的数学模拟器。在这台机器里，你可以尽情拨动“增加 1 名缝合医生”的资本开支，或者拔高“救护车到达频率”，用毫秒级别去探测这个宏大网络到底会在哪一根稻草压下时爆发连环踩踏。
